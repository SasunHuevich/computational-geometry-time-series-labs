# Лабораторная работа №1: Алгоритм Ray Casting

**Краткое описание**  
Реализация и визуальная демонстрация алгоритма **Ray Casting** (метод лучевого трассирования) для определения принадлежности точки произвольному полигону. Проект написан на языке Rust, включает модульные тесты и графический интерфейс на базе библиотеки `macroquad`.

## 🔍 Что такое Ray Casting и зачем он нужен?

Задача **«точка в полигоне»** (Point‑in‑Polygon) — одна из фундаментальных в вычислительной геометрии. Алгоритм **Ray Casting** (трассировка луча) отвечает на вопрос: лежит ли точка внутри или снаружи заданного многоугольника? Принцип прост:

1. Из точки пускается луч в произвольном направлении (обычно горизонтально вправо).
2. Подсчитывается число пересечений этого луча с рёбрами полигона.
3. Если количество пересечений **нечётно** — точка находится **внутри**, если чётно (включая 0) — **снаружи**.

Алгоритм корректен для любых простых (несамопересекающихся) многоугольников и используется в:

- **Геоинформационных системах** (определение попадания координат в район, страну);
- **Компьютерной графике** (заливка областей, выбор объектов мышью);
- **Игровой физике** (коллизии, зоны действия);
- **Робототехнике и навигации** (планирование пути, охрана периметра).

---

## ⚙️ Как работает программа

Программа выполняет следующие шаги:

1. **Задание полигона** – в коде описан пятиугольник с конкретными координатами вершин.
2. **Определение двух тестовых точек** – одна гарантированно находится внутри полигона, другая – снаружи.
3. **Реализация алгоритма Ray Casting** – функция `ray_casting` принимает точку и срез полигона, возвращает `true` (внутри) или `false` (снаружи).
4. **Графическая визуализация** – с помощью `macroquad` рисуется цветной полигон (заливка + контур), а также отображаются тестовые точки с подписями результатов.
5. **Модульные тесты** – проверяют корректность алгоритма на двух эталонных точках (успех гарантирует, что реализация работает верно).

## 🛠️ Параметры системы (Конфигурация в глобальных константах)

| Имя константы | Тип | Значение | Смысл |
| :--- | :--- | :--- | :--- |
| `POLYGON` | `[Point; 5]` | `[(100,100), (250,80), (320,180), (220,320), (80,250)]` | Вершины полигона (пятиугольник) в порядке обхода. |
| `POINT_INSIDE` | `Point` | `(180, 180)` | Точка, заведомо лежащая внутри полигона (проверено вручную). |
| `POINT_OUTSIDE` | `Point` | `(350, 250)` | Точка, заведомо лежащая снаружи полигона. |
| Размер окна | - | `640×480` (по умолчанию) | Определяется библиотекой `macroquad`. |
| Масштабирование | - | Вычисляется автоматически | Полигон масштабируется и центрируется для удобного просмотра. |

## 🖥️ Визуализация

Графическая часть реализована с использованием библиотеки **macroquad**, которая обеспечивает простой и быстрый рендеринг 2D‑геометрии. При запуске программы открывается окно с:

- **Заливкой** полигона полупрозрачным голубым цветом.
- **Контуром** полигона (синяя линия).
- **Зелёной точкой** (внутренняя) с подписью `Inside (координаты)`.
- **Красной точкой** (внешняя) с подписью `Outside (координаты)`.
- **Информационной панелью** в левом верхнем углу, где отображается логический результат проверки (`true`/`false`) для каждой точки.

Это позволяет наглядно убедиться в правильности работы алгоритма: зелёная точка всегда попадает в закрашенную область, красная — нет.
