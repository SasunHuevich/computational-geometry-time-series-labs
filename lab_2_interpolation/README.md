# Лабораторная работа №2: Интерполяция сигнала по отсчётам. Теорема Котельникова

**Краткое описание**  
Практическая демонстрация и верификация фундаментальной теоремы Найквиста — Шеннона (Котельникова) на языке Rust. Программа моделирует дискретизацию непрерывного многокомпонентного сигнала, оценивает потерю данных при нарушении условий теоремы и сравнивает математически идеальное восстановление (`sinc`) с инженерным приближением (линейная интерполяция).

---

## 🔬 Что такое теорема Котельникова и зачем она нужна?

Теорема Котельникова утверждает: любой непрерывный сигнал со спектром, ограниченным максимальной частотой $f_{max}$, может быть полностью и без потерь восстановлен по своим дискретным отсчётам, если они взяты с частотой дискретизации $f_s > 2 \cdot f_{max}$ (частота Найквиста).

Это базовый закон цифровой обработки сигналов (DSP), который определяет:
*   **Стандарты аудио** (например, 44.1 кГц для CD-дисков, чтобы передавать частоты до 22 кГц).
*   **Параметры АЦП/ЦАП** в звуковых картах, радиоприемниках и измерительных приборах.
*   **Защиту от алиасинга** (наложения спектров), который необратимо искажает данные при слишком редкой оцифровке.

---

## ⚙️ Как работает программа

Программа выполняет численный эксперимент, разделенный на следующие этапы:

1. **Генерация эталонного сигнала**: Создается аналоговая модель многокомпонентного сигнала, состоящая из трех синусоид с частотами 5 Гц, 15 Гц и $f_{max} = 20$ Гц. 
2. **Дискретизация на двух частотах**:
   *   **$f_s = 30$ Гц (НИЖЕ Найквиста)**: Нарушает условие теоремы ($30 < 40$). Провоцирует эффект алиасинга.
   *   **$f_s = 50$ Гц (ВЫШЕ Найквиста)**: Выполняет условия теоремы ($50 > 40$). Информация сохраняется.
3. **Восстановление сигнала (Интерполяция)**:
   *   **Sinc-интерполяция (Идеальный фильтр)**: Точное математическое восстановление с помощью ряда Уиттекера — Шеннона через функцию $\text{sinc}(x) = \frac{\sin(\pi x)}{\pi x}$. В коде оптимизировано оконной функцией (ограничение в 8 соседних отсчетов).
   *   **Линейная интерполяция**: Базовое соединение дискретных точек отрезками прямых.
4. **Метрический анализ ошибок**: Программа рассчитывает среднеквадратичную ошибку (MSE) и среднеквадратичное отклонение (RMSE) в процентах от пиковой амплитуды для каждого сценария.

## 🛠️ Параметры системы (Конфигурация в `main()`)

| Параметр | Значение | Смысл |
| :--- | :--- | :--- |
| `freqs` | `vec![(5.0, 1.0), (15.0, 0.7), (20.0, 0.3)]` | Состав сигнала в формате `(частота в Гц, амплитуда)`. |
| `f_max` | `20.0` Гц | Максимальная частотная компонента. Задает барьер Найквиста ($40$ Гц). |
| `fs_low` | `30.0` Гц | Частота дискретизации для демонстрации эффекта алиасинга. |
| `fs_high` | `50.0` Гц | Частота дискретизации для демонстрации точного восстановления. |
| `window` | `8` | Размер окна усечения функции `sinc` (количество учитываемых отсчетов слева и справа). |
| `test_points`| `200` | Количество контрольных точек на временном отрезке для расчета финальной ошибки. |
